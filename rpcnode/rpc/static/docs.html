📡 JSON-RPC API Overview

All RPC calls use standard JSON-RPC 2.0 over HTTP POST.

Endpoint
POST https://rpc.netlifegy.com
Also accepts WebSocket at wss://rpc.netlifegy.com/ws
Request format
Content-Type: application/json
Body: JSON-RPC 2.0 object

Example request

bash
curl -X POST https://rpc.netlifegy.com \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"eth_blockNumber","params":[],"id":1}'
json — response
{ "jsonrpc": "2.0", "id": 1, "result": "0x1a4" }
ℹ️
Batch requests are supported — send an array of request objects and receive an array of responses. See Batch Calls in the SDK section.

🚰 GYDSFaucet Contract

📦
Source: developer-tools/hardhat/contracts/GYDSFaucet.sol — Deploy with npm run deploy

A testnet faucet that drips GYDS to users on a 24-hour cooldown. Owner can adjust the drip amount and pause the faucet.

Constructor

constructor(uint256 _dripAmount, uint256 _cooldown)
Deploys faucet. _dripAmount in wei (default: 100 GYDS = 100e18). _cooldown in seconds (default: 86400).

Write Functions

request()
writenonpayable
Claim a drip of GYDS. Reverts if called within the cooldown period or if faucet balance is insufficient.
drip(address recipient)
writeonlyOwner
Owner can drip to any address, bypassing cooldown.
setDripAmount(uint256 amount)
writeonlyOwner
Change the drip amount (in wei).
pause() / unpause()
writeonlyOwner
Pause or resume the faucet.

Read Functions

nextRequestAt(address user) → uint256
view
Timestamp when user can next call request().
balance() → uint256
view
Current GYDS balance held by the faucet.

Usage example

javascript
import { ethers } from 'ethers';

const abi = [
  'function request() external',
  'function nextRequestAt(address) view returns (uint256)',
  'function balance() view returns (uint256)',
];
const provider = new ethers.JsonRpcProvider('https://rpc.netlifegy.com');
const faucet   = new ethers.Contract(FAUCET_ADDRESS, abi, signer);

// Check cooldown
const next = await faucet.nextRequestAt(userAddress);
if (Date.now() / 1000 >= Number(next)) {
  const tx = await faucet.request();
  await tx.wait();
  console.log('Drip received!');
}

💡 SDK Examples

Send a GYDS transfer

typescript
import { ethers } from 'ethers';

const provider = new ethers.JsonRpcProvider('https://rpc.netlifegy.com');
const wallet   = new ethers.Wallet(PRIVATE_KEY, provider);

const tx = await wallet.sendTransaction({
  to:      '0xRecipientAddress',
  value:   ethers.parseEther('10'),  // 10 GYDS
  chainId: 13370,
});

const receipt = await tx.wait();
console.log(`Confirmed in block ${receipt.blockNumber}`);

Read validator set

typescript
import { GYDSClient } from '@gydschain/sdk';

const client = new GYDSClient({ rpcUrl: 'https://rpc.netlifegy.com' });
const { validators, epoch } = await client.getValidatorSet();

for (const v of validators) {
  console.log(`[${v.index}] ${v.address} — ${v.status}`);
}
console.log(`Current epoch: ${epoch}`);

Deploy a contract with Hardhat

javascript — scripts/deploy.js
const [deployer] = await ethers.getSigners();

const faucet = await Faucet.deploy(
  ethers.parseEther('100'),  // drip 100 GYDS
  86400                      // 24h cooldown
);
await faucet.waitForDeployment();
console.log('Faucet deployed to:', await faucet.getAddress());

⚡ SDK — Batch Calls

Send multiple RPC calls in a single HTTP request for maximum efficiency.

typescript
import { GYDSClient } from '@gydschain/sdk';

const client = new GYDSClient({ rpcUrl: 'https://rpc.netlifegy.com' });

const [blockNum, gasPrice, validators] = await client.batch([
  { method: 'eth_blockNumber',    params: [] },
  { method: 'eth_gasPrice',        params: [] },
  { method: 'gyds_validatorSet',   params: [] },
]);

console.log('Block:',      parseInt(blockNum.result,   16));
console.log('Gas price:', parseInt(gasPrice.result,   16), 'wei');
console.log('Validators:',validators.result.validators);

Raw HTTP batch request

bash
curl -X POST https://rpc.netlifegy.com \
  -H "Content-Type: application/json" \
  -d '[
    {"jsonrpc":"2.0","method":"eth_blockNumber","params":[],"id":1},
    {"jsonrpc":"2.0","method":"gyds_validatorSet","params":[],"id":2},
    {"jsonrpc":"2.0","method":"gyds_nodeInfo","params":[],"id":3}
  ]'

🖥️ Node Types

🔷 RPC Node
Public-facing JSON-RPC endpoint. Handles eth_* calls, serves the dashboard UI, proxies requests to the chain. One per public-facing server.
🟢 Validator Node
Signs and proposes blocks. Requires 32,000 GYDS stake. Runs P2P gossip protocol. Private — never expose port 8545 publicly.
🔵 Full Node
Stores full chain history, validates all transactions, maintains mempool and receipts. Used for indexing and block explorers.
🟣 Lite Node
Lightweight SPV node. Verifies block headers only. Suitable for mobile or resource-constrained environments.

BroadcastBlock, BroadcastTx, peer discovery, checkpoint sync via gossip protocol
Block Rewards
2 GYDS per block. Halving at 105M blocks. UptimeTracker monitors missed slots.

🛡️ Security & Firewall

UFW firewall rules
bash
ufw default deny incoming
ufw allow ssh
ufw allow 443/tcp   # HTTPS
ufw allow 80/tcp    # HTTP
ufw allow 8545/tcp  # RPC (restrict to trusted IPs)
ufw allow 30303     # P2P
ufw enable
WireGuard VPN
All inter-node communication (validator ↔ validator, validator ↔ full node) runs over a private WireGuard VPN mesh.
bash
sudo bash setup-wireguard.sh
⚠️
Never expose your validator node's RPC port (8545) to the public internet. Use the RPC node as a public-facing proxy, keeping your validators behind the VPN and firewall.
